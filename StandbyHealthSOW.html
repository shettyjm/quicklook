Executive summary

StandbyHealth is a physician-supervised AI second opinion platform. NRI families in the US upload medical records from India — discharge summaries, lab reports, DICOM imaging — and receive a structured physician opinion within 24 hours. A licensed US physician reviews an AI-generated draft, edits as needed, and delivers a signed PDF.

The platform is HIPAA-compliant by design. All PHI-touching infrastructure runs under Azure's HIPAA Business Associate Agreement. AI assists; physicians decide.

🎯 Primary market: NRI families coordinating care for parents in India across Cardiology, Oncology, and Neurology.

User roles and onboarding flows
Five user types, each with dedicated signup, onboarding, and workflow experiences.

👤 Patient (Priya)

Email/Google signup, HIPAA consent, profile completion, multi-step intake wizard with drag-and-drop upload, payment, real-time status tracker, opinion delivery, share with family, follow-up re-upload

👨‍👩‍👧 Family member (Rahul)

Invite-based signup from patient, scoped permissions (view reports, receive alerts), CD upload tool, WhatsApp status updates, Hindi/English toggle, downloadable opinion PDF

🩺 Physician (Dr. Roger)

Credentialed login with MFA, first-login dashboard walkthrough, case queue sorted by urgency, AI summary review + edit, sign opinion, video consult, mobile app push alerts

🔬 Radiologist (Dr. Andrew)

Credentialed login + MFA, OHIF viewer onboarding, split-pane DICOM review, annotation + measurement tools, structured radiology opinion form, voice-to-text notes

⚙️ Admin

Internal login, case volume dashboard with sparklines, revenue trends, physician utilization heatmap, AI quality metrics (zero-edit rate), cost-per-case tracking, audit log access, exportable reports

Assumptions and exclusions

Included in scope

  • All items listed in Phases 1-3 including UI/UX design work
  • Design system, component library, and responsive layouts
  • User onboarding flows for all roles: patient, physician, radiologist, family, admin
  • Code for all microservices, frontend pages, and infrastructure
  • HIPAA-compliant architecture throughout
  • Unit tests (Vitest) and E2E tests (Playwright) for all features
  • Terraform IaC for all Azure resources
  • Accessibility audit (WCAG 2.1 AA) in Phase 3
  • Documentation and knowledge transfer

Excluded from scope

  • FDA SaMD regulatory filing — AI positioned as clinical decision support, not diagnosis
  • State medical board licensing for physicians (client responsibility)
  • HIPAA BAA negotiations with vendors beyond Azure (client responsibility)
  • Physician compensation and malpractice insurance (client responsibility)
  • Marketing website, SEO, paid acquisition
  • iOS/Android App Store submission fees and review process
  • Indian hospital EHR API access agreements (client to negotiate with Apollo, Fortis)

Assumptions

  • Azure subscription with active HIPAA BAA provided by client
  • Stripe account and KYC verification completed by client
  • Sample medical records provided for AI prompt tuning
  • Client provides access to at least one Indian hospital for Phase 3 EHR pilot
